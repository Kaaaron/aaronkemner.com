TechAnim in Unreal: AnimNotify and AnimNotifyState

Among the most powerful tools that Unreal Engine offers out-of-the-box are AnimNotifies and AnimNotifyStates. These two features allow us to execute any logic we might want at a specific point in an animation. That might be firing of VFX at the feet of our character each time they take a step, or playing an audio cue while they are charging up an attack, or something else entirely.

They are one of the most-used ways of having animation directly interface with the rest of the game and I can pretty much guarantee you that any Unreal Engine Project of quality uses them in some capacity. They are hugely flexible tools, for anyone touching animation assets, be that Animators, Tech-Artists, or Engineers, so knowing their ins and outs is key to creating great games and workflows.

So lets take a deeper look at what this means.

Using AnimNotifies

When talking about these, we are really talking about instances of classes that derive either from UAnimNotify or UAnimNotifyState, which are both organized into an Array ofFAnimNotifyTrack on any Asset deriving from UAnimSequenceBase.

This means you can add AnimNotifies to UAnimSequences and AnimComposites like UAnimMontage, but not other animation assets like UBlendSpace. Nonetheless, Blendspaces allow triggering of AnimNotifies for samples within the Blendspace as well.

API Overview

AnimNotifies and AnimNotifyStates have a very similar API, especially All of these are always called on the GameThread after evaluation has completed.

Runtime

// ------- Only UAnimNotify  -----------
// Called when resolving the NotifyQueue. 
virtual void Notify(USkeletalMeshComponent* MeshComp, UAnimSequenceBase* Animation, const FAnimNotifyEventReference& EventReference);
// Called for branching point Notifies only. Will call Notify() if not implemented.
virtual void BranchingPointNotify(FBranchingPointNotifyPayload& BranchingPointPayload);

// ------- Only UAnimNotifyState -------
// Called before NotifyTick is called the first time, signaling the start of the Notify window.
virtual void NotifyBegin(USkeletalMeshComponent * MeshComp, UAnimSequenceBase * Animation, float TotalDuration, const FAnimNotifyEventReference& EventReference);
// Called each frame after NotifyBegin has been called once and before NotifyEnd is called.
virtual void NotifyTick(USkeletalMeshComponent * MeshComp, UAnimSequenceBase * Animation, float FrameDeltaTime, const FAnimNotifyEventReference& EventReference);
// Called after NotifyTick has been called for the last time, signaling the end of the Notify window.
virtual void NotifyEnd(USkeletalMeshComponent * MeshComp, UAnimSequenceBase * Animation, const FAnimNotifyEventReference& EventReference);
// Called for branching point Notifies only. Will call NotifyBegin() if not implemented.
virtual void BranchingPointNotifyBegin(FBranchingPointNotifyPayload& BranchingPointPayload);
// Called for branching point Notifies only. Will call NotifyTick() if not implemented.
virtual void BranchingPointNotifyTick(FBranchingPointNotifyPayload& BranchingPointPayload, float FrameDeltaTime);
// Called for branching point Notifies only. Will call NotifyEnd() if not implemented.
virtual void BranchingPointNotifyEnd(FBranchingPointNotifyPayload& BranchingPointPayload);

// ------- UAnimNotify and UAnimNotifyState -----------

Special case: SyncMarkers

SyncMarkers are one way in Unreal to coordinate multiple animations you want to blend/transition between. In code, these are represented as FAnimSyncMarker, and they have the huge disadvantage that they have no point where one might hook into the system to customize/extend UEs animation syncing. There is also no such notion as a “SyncMarkerState” - they are just markers at a single frame, without a notion of “duration”. They contain a Name, and UE will try to sync them with other markers of the same name, and of course a position at which time in the animation they are positioned.

While SyncMarkers are not really the aim of this post, due to the lack of extendability, they are definitely a hugely useful tool to have when working with Animation in Unreal. So if you haven’t had the chance to, I recommend having a look at the docs.
